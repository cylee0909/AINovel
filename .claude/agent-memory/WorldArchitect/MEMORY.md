# 世界观架构师记忆

## 项目模式
- 历史穿越类小说需要平衡真实历史与虚构元素
- 爽文向世界观需要明确的“金手指”设定和成长路径
- 战国时期资料丰富，需要准确的历史时间节点和地理格局

## 文件生成规范
- `世界观.md`：约1000字，包含5个核心部分
- `矛盾.md`：约500字，包含4类冲突和伏笔清单
- 输出必须使用markdown代码块格式

## 历史类世界观要点
1. **时间准确性**：公元前350年是关键节点，秦孝公求贤、商鞅变法前夕
2. **地理真实性**：七雄疆域、重要关隘、都城位置必须准确
3. **力量体系**：历史爽文的力量体系是知识、科技、制度创新
4. **科技树限制**：每个科技引入都需要合理的限制和代价
5. **暗线设计**：在真实历史中加入神秘元素作为伏笔（九鼎、昆仑遗民等）

## 冲突设计模式
1. **核心冲突**：历史必然性 vs 个人意志
2. **阵营冲突**：至少3组相互制约的势力
3. **理念冲突**：无绝对善恶，只有立场不同
4. **伏笔设计**：3-5个后期引爆的核心矛盾

## 用户偏好
- 喜欢详细的历史背景设定
- 重视逻辑自洽和现实可行性
- 需要明确的成长路径和爽点设计
- 期待复杂的阵营关系和理念冲突
